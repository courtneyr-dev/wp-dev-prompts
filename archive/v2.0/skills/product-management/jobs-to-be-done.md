<summary>
Structured framework for understanding WordPress user needs through functional, social, and emotional jobs.
</summary>

<knowledge>
## When to Use

Apply this framework when:
- Planning a new WordPress plugin or theme
- Evaluating feature requests for existing products
- Understanding why users choose competitors
- Prioritizing your product roadmap

## WordPress User Segments

### Site Owners/Administrators
- **Functional**: Keep site running, secure, and fast
- **Social**: Appear professional and credible
- **Emotional**: Peace of mind, control over their presence

### Content Creators/Editors
- **Functional**: Publish content efficiently
- **Social**: Build audience and authority
- **Emotional**: Creative satisfaction, reduced frustration

### Developers
- **Functional**: Build and maintain sites efficiently
- **Social**: Deliver quality work to clients
- **Emotional**: Pride in craft, reduced tedium

### Agency Owners
- **Functional**: Manage multiple sites profitably
- **Social**: Retain clients, win referrals
- **Emotional**: Business growth confidence

## Framework Structure

### 1. Customer Jobs

**Functional Jobs** — What task must they complete?
```
When [situation], I want to [action] so I can [outcome].
```

WordPress examples:
- "When launching a new product, I want to create a landing page so I can capture leads"
- "When my site slows down, I want to identify the cause so I can fix it quickly"
- "When updating plugins, I want to know if anything will break so I can update confidently"

**Social Jobs** — How do they want to be perceived?
```
I want [audience] to see me as [perception].
```

WordPress examples:
- "I want visitors to see my site as professional and trustworthy"
- "I want clients to see me as technically competent"
- "I want my team to see me as organized and in control"

**Emotional Jobs** — What feeling do they seek/avoid?
```
I want to feel [positive emotion] / avoid feeling [negative emotion].
```

WordPress examples:
- "I want to feel confident my site won't get hacked"
- "I want to avoid the anxiety of not knowing if my backups work"
- "I want to feel proud of my site's design"

### 2. Pains

**Challenges** — What blocks progress?
- Plugin conflicts causing white screens
- Slow page loads losing visitors
- Security vulnerabilities they don't understand
- Updates breaking functionality

**Costliness** — What costs too much time/money/effort?
- Hours spent on manual backups
- Expensive hosting for simple sites
- Developer fees for small changes
- Time lost to spam management

**Common Mistakes** — What do they get wrong?
- Installing too many plugins
- Ignoring update notifications
- Weak passwords and no 2FA
- No staging environment for testing

**Unresolved Problems** — What's still broken?
- No easy way to test updates safely
- Difficult to compare plugin options
- Hard to diagnose performance issues
- Complex migration processes

### 3. Gains

**Expectations** — How should the solution perform?
- Faster than current workflow
- Fewer steps to accomplish tasks
- Works without reading documentation
- Integrates with existing tools

**Savings** — What would they value saving?
- 2 hours/week on maintenance tasks
- Reduced hosting costs
- Fewer support tickets
- Less developer dependency

**Adoption Factors** — What encourages use?
- One-click installation
- Familiar WordPress UI patterns
- Free tier to evaluate
- Active support community

**Life Improvements** — What makes work better?
- Spend time on content, not tech
- Sleep well knowing site is secure
- Impress clients with fast delivery
- Scale without proportional effort

## Output Template

```markdown
## Jobs to Be Done Analysis: [Product Name]

### Target Segment
[Primary user type and context]

### Functional Jobs
1. When [situation], I want to [action] so I can [outcome]
2. ...

### Social Jobs
1. I want [audience] to see me as [perception]
2. ...

### Emotional Jobs
1. I want to feel [emotion] / avoid [emotion]
2. ...

### Top 3 Pains
1. [Pain point with impact]
2. ...

### Top 3 Gains Sought
1. [Desired outcome with value]
2. ...

### Opportunity Statement
[How your product addresses the most important job]
```
</knowledge>

<best_practices>
- Interview actual WordPress users, not just developers
- Focus on the job, not the solution they request
- Identify the "hiring criteria" — why they'd choose your product
- Map competing solutions to the same jobs
</best_practices>
