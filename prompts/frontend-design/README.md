# Frontend Design Prompts

AI prompts for generating accessible, responsive UI components with consistent design language.

## Prompts

| Prompt | Description | Use Case |
|--------|-------------|----------|
| `component-design.md` | Generate accessible UI components | Creating new components with HTML, CSS, React |
| `style-language.md` | Define aesthetic style specifications | Establishing design tokens for a visual style |
| `theme-vs-plugin-styles.md` | Determine style ownership | Deciding where styles belong in WordPress |
| `responsive-behavior.md` | Define responsive component behavior | Mobile-first responsive patterns |
| `accessibility-presets.md` | Generate comprehensive a11y specs | WCAG compliance, ARIA patterns |

## Quick Start

### Component Design

Generate a complete, accessible component:

```markdown
Create a card component with:
- Thumbnail image
- Title (clickable)
- Description (max 3 lines)
- Category badge
- Read more link
```

### Style Language

Define a complete design system:

```markdown
Generate a "Nordic Minimalist" style language with:
- Color palette (light, muted, natural tones)
- Typography scale
- Spacing rhythm
- Border and shadow treatments
```

### Responsive Behavior

Specify how components adapt:

```markdown
Define responsive behavior for a navigation menu:
- Desktop: horizontal menu with dropdowns
- Tablet: collapsed hamburger with slide-out
- Mobile: full-screen overlay
```

## Style Languages

The `style-language.md` prompt supports 10 aesthetic styles:

| Style | Description |
|-------|-------------|
| Art Deco | Geometric patterns, gold accents, luxury feel |
| Brutalist | Raw, bold, high-contrast, intentionally rough |
| Nordic/Scandinavian | Minimal, light, functional, natural |
| Neo-Classical | Elegant, symmetrical, refined |
| Biophilic | Nature-inspired, organic, calming |
| Maximalist | Bold, layered, expressive, colorful |
| Zen/Japanese | Simple, balanced, intentional whitespace |
| Cyberpunk | Neon, dark, futuristic, high-tech |
| Bauhaus | Functional, geometric, primary colors |
| Retro-Futurism | Vintage sci-fi, optimistic, chromatic |

## Integration with Storybook

These prompts pair with the Storybook configuration in `ui/storybook/`:

1. Generate component with `component-design.md`
2. Apply style with `style-language.md`
3. Preview in Storybook with theme switcher
4. Test accessibility with addon-a11y

## WordPress Integration

### theme.json Mapping

Style tokens can be exported to WordPress theme.json:

```json
{
  "settings": {
    "color": {
      "palette": [
        { "slug": "primary", "color": "#D4AF37" },
        { "slug": "secondary", "color": "#1a1a2e" }
      ]
    }
  }
}
```

### CSS Custom Properties

All styles use CSS custom properties for easy theming:

```css
:root {
  --wp--preset--color--primary: #D4AF37;
  --wp--preset--spacing--medium: 1.5rem;
}
```

## Output Format

Components generated by these prompts include:

1. **HTML Structure** - Semantic, accessible markup
2. **CSS/SCSS** - Responsive styles with custom properties
3. **React/JSX** - Component implementation (optional)
4. **ARIA Specification** - Roles, states, properties
5. **Keyboard Behavior** - Focus management, shortcuts
6. **Playwright Tests** - Accessibility and interaction tests
